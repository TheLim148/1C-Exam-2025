# Билет №1
## Устные задания
**1. Опишите основные объекты метаданных в системе «1С:Предприятие» и их назначение.**
Основные объекты метаданных — это логические элементы конфигурации, которые определяют структуру данных и поведение системы. К ключевым объектам относятся:
- Справочники — хранят перечни сущностей (товары, сотрудники, контрагенты). Используются для постоянных данных, к которым обращаются документы.
- Документы — фиксируют события (поступление товаров, продажа, приём на работу). Могут иметь табличные части, номера, дату, механизм проведения.
- Регистры накопления — хранят остатки и обороты (склады, продажи). Документы формируют в них движения.
- Регистры сведений — хранят произвольные состояния и значения (например, цены, настройки).
- Регистры бухгалтерии и расчёта — используются в специализированных конфигурациях.
- Общие модули — содержат процедуры и функции, доступные другим объектам.
- Формы — создают интерфейс для работы пользователя (списки, карточки объектов).
- Планы видов характеристик — определяют возможные свойства объектов.
- Подсистемы — определяют структуру интерфейса и организацию команд.
- Роли — задают права пользователей.
Все объекты метаданных вместе формируют конфигурацию — логику и структуру работы прикладного решения.

---
**2. Объясните процесс проведения документа и задействованные модули.**
Проведение документа — это механизм, который фиксирует влияние документа на данные системы (движения по регистрам, проверки, расчёты).
Основные этапы:
1. Проверка заполненности — перед проведением документ проверяет обязательные реквизиты, табличные части, корректность данных.
2. Выполнение модуля объекта (МодульОбъекта) — здесь реализуется основная логика проведения:
- формируются движения по регистрам накопления/сведений,
- выполняются расчёты,
- проверяются условия бизнес-логики.
3. Запись движений в регистры — система фиксирует изменения состояния (остатки, цены, статусы).
4. Возврат результата — успешное проведение или сообщение об ошибке.
Дополнительно может участвовать:
- Модуль менеджера документа (общие операции с коллекцией документов);
- Подписки на события — позволяют дополнять проведение без изменения типовой логики;
- Общие модули — используются для вынесения общих алгоритмов.

В результате проведения документ становится частью последовательности событий системы и начинает влиять на отчёты, регистры и аналитику.
